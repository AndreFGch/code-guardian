# Publishing Notes

Code Guardian should be developed first as an internal skill.

Before publishing:

1. Remove personal paths.
2. Add a clear license.
3. Add examples.
4. Add screenshots or sample reports if useful.
5. Validate that the skill does not include secrets.
6. Validate that the skill does not copy content from external skills.
7. Add installation instructions.
8. Add limitations.
9. Add a security disclaimer.
10. Tag the first release.

## Possible GitHub description

A Claude Code skill for repository health, safety, architecture and scope review before making changes.

## Possible LinkedIn description

I built Code Guardian, a Claude Code skill that reviews repository health, security risks, structure and change scope before allowing an AI coding agent to modify files. It is designed for safer, smaller and more auditable development workflows.
